# MistRelay 更新文档

本文档是 MistRelay Windows 桌面客户端正式版更新系统的运维发布手册，适用于发布维护、更新链路排查和手动验证。

当前基准版本：`1.0.4`。

## 1. 更新系统概览

MistRelay Windows 桌面客户端使用 GitHub Releases 作为更新分发源。

客户端启动或用户手动检查更新时，会读取 `version.json` 中的更新配置：

- `release_feed_url`: GitHub Releases API 地址。
- `release_tag_prefix`: 发布 tag 前缀，当前为 `desktop-qt-v`。
- `verify_key`: Ed25519 更新公钥，base64 编码。

客户端更新流程：

1. 请求 GitHub Releases API。
2. 查找最新 `desktop-qt-v*` Release。
3. 下载 `qt-latest.json`。
4. 下载 `qt-latest.json.sig` 并使用公钥验签。
5. 判断是否存在可用文件级补丁。
6. 优先下载并应用 `qt-patch-<from>-to-<to>.zip`。
7. 如果补丁不存在、不适用或失败，则使用整包安装器更新。

## 2. Release 资产

正式 Release 至少应包含：

| 资产 | 说明 |
| --- | --- |
| `mistrelay-v<version>-setup.exe` | Windows NSIS 安装器，整包更新兜底使用。 |
| `qt-latest.json` | 更新清单，记录版本、安装器和补丁包信息。 |
| `qt-latest.json.sig` | `qt-latest.json` 的 Ed25519 detached signature。 |

当存在上一版本 tag 且 CI 配置了 `QT_UPDATE_PRIVATE_KEY` 时，还会生成：

| 资产 | 说明 |
| --- | --- |
| `qt-patch-<from>-to-<to>.zip` | 文件级热更补丁包，按变更文件完整内容打包。 |

示例：

```text
mistrelay-v1.0.4-setup.exe
qt-latest.json
qt-latest.json.sig
qt-patch-1.0.1-to-1.0.4.zip
```

## 3. 更新清单结构

`qt-latest.json` 使用 UTF-8 JSON。

核心字段：

```json
{
  "version": "1.0.4",
  "notes": "",
  "pub_date": "2026-04-24T00:00:00Z",
  "platforms": {
    "windows-x86_64": {
      "url": "https://github.com/qianlong520/MistRelay/releases/download/desktop-qt-v1.0.4/mistrelay-v1.0.4-setup.exe",
      "sha256": "...",
      "size": 123456
    }
  },
  "patches": {
    "windows-x86_64": {
      "from_version": "1.0.1",
      "to_version": "1.0.4",
      "url": "https://github.com/qianlong520/MistRelay/releases/download/desktop-qt-v1.0.4/qt-patch-1.0.1-to-1.0.4.zip",
      "sha256": "...",
      "size": 12345,
      "signature": "...",
      "requires_restart": false,
      "reloadable_paths": [
        "mistrelay_qt/qml/Main.qml"
      ]
    }
  }
}
```

说明：

- `platforms.windows-x86_64` 是整包安装器信息，必须保留。
- `patches.windows-x86_64` 是可选字段，只有存在匹配当前版本的补丁时才使用。
- `from_version` 必须等于客户端当前版本，否则客户端会忽略该补丁。
- `to_version` 必须等于清单 `version`。
- `sha256` 和 `size` 用于下载后校验。
- `signature` 是补丁包内 `patch-manifest.json` 的签名。
- `requires_restart` 表示补丁中是否包含需要重启替换的文件。
- `reloadable_paths` 记录可立即替换的资源路径。

## 4. 补丁包结构

`qt-patch-<from>-to-<to>.zip` 是签名 zip 包。

结构：

```text
qt-patch-1.0.1-to-1.0.4.zip
├─ patch-manifest.json
└─ files/
   └─ <relative-path>
```

`patch-manifest.json` 示例：

```json
{
  "format": 1,
  "from_version": "1.0.1",
  "to_version": "1.0.4",
  "created_at": "2026-04-24T00:00:00Z",
  "download_url": "https://github.com/qianlong520/MistRelay/releases/download/desktop-qt-v1.0.4/qt-patch-1.0.1-to-1.0.4.zip",
  "requires_restart": false,
  "reloadable_paths": ["mistrelay_qt/qml/Main.qml"],
  "files": [
    {
      "path": "mistrelay_qt/qml/Main.qml",
      "action": "replace",
      "sha256": "...",
      "size": 1234
    }
  ],
  "signature": "..."
}
```

安全约束：

- `path` 必须是相对路径。
- 禁止绝对路径。
- 禁止 `..` 路径穿越。
- 禁止反斜杠路径分隔符。
- `action` 仅支持 `add`、`replace`、`delete`。
- 每个文件必须匹配清单中的 `sha256` 和 `size`。

## 5. 整包更新

整包更新使用 NSIS 安装器：

```text
mistrelay-v<version>-setup.exe
```

客户端下载后会：

1. 校验安装器 `sha256`。
2. 校验安装器 `size`。
3. 生成 PowerShell 更新脚本。
4. 等待当前客户端进程退出。
5. 使用 `/S` 参数静默安装。
6. 安装成功后重启客户端。

整包更新适用于：

- 首次安装。
- 补丁不存在。
- 补丁版本不匹配。
- 补丁验签失败。
- 补丁应用失败。
- 大版本或运行时依赖变化较大。

## 6. 文件级热更

文件级热更优先于整包更新。

客户端只有在满足以下条件时才使用补丁：

- `qt-latest.json` 验签通过。
- `patches.windows-x86_64` 存在。
- `from_version` 等于当前客户端版本。
- `to_version` 等于目标版本。
- 补丁包下载地址、`sha256` 和 `size` 完整。

补丁应用策略：

- QML、主题、图片、JSON 等资源文件可直接备份并替换。
- `exe`、`dll`、`pyd`、Python runtime zip 等文件视为需要重启替换。
- 需要重启替换的文件会写入 `updates/<version>/pending-files`。
- 客户端生成外部 PowerShell 脚本，等待当前进程退出后复制待替换文件并重启客户端。

回滚策略：

- 应用补丁前，客户端会把被替换文件备份到 `update-backups/<version>`。
- 任一步失败会立即恢复已替换文件。
- 补丁成功后保留上一版本备份，用于后续排查或人工恢复。

## 7. GitHub Actions 发布流程

工作流文件：

```text
.github/workflows/build-windows-desktop-qt.yml
```

触发方式：

- 推送 tag：`desktop-qt-v*`
- 手动 `workflow_dispatch`

CI 步骤：

1. Checkout，并拉取完整 tag 历史。
2. 解析版本号和 tag。
3. 安装 Python 依赖。
4. 安装 NSIS。
5. 同步 `version.json` 中的版本号、通道和更新公钥。
6. 运行发布前检查。
7. 构建 PyInstaller onedir 和 NSIS 安装器。
8. 查找上一个 `desktop-qt-v*` tag。
9. 基于上一个 tag 构建旧版 PyInstaller onedir。
10. 对比旧版和新版 onedir，生成 `qt-patch-<from>-to-<to>.zip`。
11. 生成 `qt-latest.json` 和 `qt-latest.json.sig`。
12. 校验清单、签名、安装器 hash 和补丁包。
13. 创建 GitHub Release 并上传资产。

## 8. 发布前准备

### 8.1 检查更新密钥

必须配置：

- `QT_UPDATE_PRIVATE_KEY`: Ed25519 签名私钥，base64 编码。

建议配置：

- `QT_UPDATE_VERIFY_KEY`: Ed25519 公钥，base64 编码。

如果没有私钥：

- CI 会跳过 `qt-latest.json` 生成。
- 客户端无法自动发现该版本更新。
- 文件级补丁也不会生成。

### 8.2 本地检查

发布前运行：

```powershell
.\.venv\Scripts\python.exe scripts/dev_env.py release-check --require-update-key
```

该命令会检查：

- `version.json` 版本号格式。
- 更新源配置。
- 更新公钥是否存在。
- Python 编译。
- 单元测试。
- 补丁生成与校验测试。

### 8.3 构建检查

如果本机安装了 NSIS，可运行：

```powershell
.\.venv\Scripts\python.exe scripts/dev_env.py doctor --build
.\.venv\Scripts\python.exe scripts/dev_env.py build --clean
```

如果只验证 PyInstaller：

```powershell
.\.venv\Scripts\python.exe scripts/dev_env.py build --clean --skip-installer
```

## 9. 正式发布步骤

1. 更新 `version.json`：

```json
{
  "version": "1.0.4",
  "channel": "stable"
}
```

2. 运行发布前检查：

```powershell
.\.venv\Scripts\python.exe scripts/dev_env.py release-check --require-update-key
```

3. 提交代码：

```powershell
git add .
git commit -m "Release desktop v1.0.4"
```

4. 创建 tag：

```powershell
git tag desktop-qt-v1.0.4
```

5. 推送：

```powershell
git push origin main
git push origin desktop-qt-v1.0.4
```

6. 在 GitHub Actions 查看构建状态。

7. 在 GitHub Release 确认资产完整。

## 10. 手动生成补丁包

如果需要本地验证补丁生成，可准备两个 PyInstaller onedir 目录：

```text
build/old/MistRelay
build/new/MistRelay
```

生成补丁：

```powershell
.\.venv\Scripts\python.exe scripts/release_manifest.py generate-patch `
  --from-version 1.0.3 `
  --to-version 1.0.4 `
  --old-dir build/old/MistRelay `
  --new-dir build/new/MistRelay `
  --output dist/windows/qt-patch-1.0.1-to-1.0.4.zip `
  --download-url https://github.com/qianlong520/MistRelay/releases/download/desktop-qt-v1.0.4/qt-patch-1.0.1-to-1.0.4.zip `
  --private-key $env:QT_UPDATE_PRIVATE_KEY
```

校验补丁：

```powershell
.\.venv\Scripts\python.exe scripts/release_manifest.py verify-patch `
  --patch dist/windows/qt-patch-1.0.1-to-1.0.4.zip `
  --public-key $env:QT_UPDATE_VERIFY_KEY `
  --from-version 1.0.3 `
  --to-version 1.0.4
```

## 11. 手动生成更新清单

生成清单和签名：

```powershell
.\.venv\Scripts\python.exe scripts/release_manifest.py generate `
  --version 1.0.4 `
  --installer dist/windows/mistrelay-v1.0.4-setup.exe `
  --download-url https://github.com/qianlong520/MistRelay/releases/download/desktop-qt-v1.0.4/mistrelay-v1.0.4-setup.exe `
  --output dist/windows/qt-latest.json `
  --signature-output dist/windows/qt-latest.json.sig `
  --patch dist/windows/qt-patch-1.0.1-to-1.0.4.zip `
  --patch-url https://github.com/qianlong520/MistRelay/releases/download/desktop-qt-v1.0.4/qt-patch-1.0.1-to-1.0.4.zip `
  --patch-from-version 1.0.3 `
  --private-key $env:QT_UPDATE_PRIVATE_KEY
```

校验清单：

```powershell
.\.venv\Scripts\python.exe scripts/release_manifest.py verify `
  --version 1.0.4 `
  --installer dist/windows/mistrelay-v1.0.4-setup.exe `
  --download-url https://github.com/qianlong520/MistRelay/releases/download/desktop-qt-v1.0.4/mistrelay-v1.0.4-setup.exe `
  --manifest dist/windows/qt-latest.json `
  --signature dist/windows/qt-latest.json.sig `
  --public-key $env:QT_UPDATE_VERIFY_KEY
```

## 12. 常见问题排障

### 12.1 客户端找不到更新

检查：

- Release tag 是否符合 `desktop-qt-v<semver>`。
- Release 是否为 draft。
- Release 是否包含 `qt-latest.json` 和 `qt-latest.json.sig`。
- `version.json.release_feed_url` 是否指向当前仓库。
- 客户端当前版本是否低于清单版本。

### 12.2 更新清单验签失败

检查：

- `QT_UPDATE_PRIVATE_KEY` 和 `QT_UPDATE_VERIFY_KEY` 是否是一组密钥。
- 打包产物中的 `version.json.verify_key` 是否正确。
- `qt-latest.json` 上传后是否被手动修改。
- `qt-latest.json.sig` 是否来自同一次清单生成。

### 12.3 CI 没有生成补丁包

可能原因：

- 未配置 `QT_UPDATE_PRIVATE_KEY`。
- 仓库没有上一版 `desktop-qt-v*` tag。
- 上一版 tag 无法构建 PyInstaller onedir。
- `generate-patch` 或 `verify-patch` 失败。

处理方式：

- 查看 GitHub Actions 的 `Generate file patch` 步骤日志。
- 如果补丁生成失败但整包和清单存在，可先使用整包更新。
- 修复旧版构建问题后重新运行 workflow。

### 12.4 补丁不生效

检查：

- `patches.windows-x86_64.from_version` 是否等于客户端当前版本。
- `patches.windows-x86_64.to_version` 是否等于清单 `version`。
- 补丁 zip 的 `sha256` 和 `size` 是否匹配。
- `patch-manifest.json` 是否包含有效签名。
- 补丁路径是否被安全校验拒绝。

### 12.5 二进制文件更新后需要重启

这是预期行为。

以下文件运行中通常不能安全覆盖：

- `.exe`
- `.dll`
- `.pyd`
- Python runtime zip

客户端会把这些文件写入 `updates/<version>/pending-files`，并通过外部脚本在客户端退出后复制到安装目录，再重启客户端。

### 12.6 NSIS 或 makensis 缺失

本地构建安装器前运行：

```powershell
.\.venv\Scripts\python.exe scripts/dev_env.py doctor --build
```

如果提示缺少 `makensis`：

- 安装 NSIS。
- 确认 `makensis.exe` 在 PATH 中，或安装在默认路径。

### 12.7 Release 资产缺失

正式 Release 至少需要：

- `mistrelay-v<version>-setup.exe`
- `qt-latest.json`
- `qt-latest.json.sig`

如果缺少 `qt-patch-*.zip`，客户端仍可使用整包更新。

## 13. 运维建议

- 每次正式发布前运行 `release-check --require-update-key`。
- 不要手动编辑已签名的 `qt-latest.json`。
- 不要复用错误版本的 `qt-latest.json.sig`。
- 发布后检查 Release 资产是否完整。
- 对包含运行时依赖、PySide6、Python 版本或安装器逻辑变化的版本，优先预期整包更新。
- 文件级补丁适合 QML、主题、资源、小范围 Python 相关文件和配置资源变化，但仍需保留整包兜底。
